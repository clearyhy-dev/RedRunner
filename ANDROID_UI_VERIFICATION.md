# Android UI 验收清单

## 当前代码侧已完成
- 已统一 `CanvasScaler` 的手机参考分辨率为 `1080x1920`
- 已新增通用 `SafeAreaFitter`
- 已将 `UIScreen` 根节点纳入全屏拉伸修复
- 已将 Google 登录入口改为安全区内的受控布局
- 已关闭 Android 上不需要的桌面鼠标/光标逻辑

## 建议回归机型
- 720x1600 左右的低端安卓机
- 1080x1920 或 1080x2340 的常见安卓机
- 带刘海/挖孔屏的安卓机

## 进入 Unity 后先执行
1. 打开主场景 `Play`
2. 执行 `Meow Runner/Android 适配/配置当前场景 Canvas 自适应`
3. 执行 `Meow Runner/Android 适配/拉伸当前场景 UIScreen 到全屏`
4. 如有顶部或底部容器需要避让刘海/手势区，选中对象后执行 `Meow Runner/Android 适配/给选中对象添加 SafeAreaFitter`
5. 执行 `Meow Runner/Android 适配/检查当前场景手机适配问题`
6. 执行 `Meow Runner/Android 适配/设置安卓竖屏（推荐）`

## 真机检查项

### 首页 / 开始页
- 主按钮完整显示，不超出屏幕边界
- Google 登录入口完整显示
- 文本没有被裁切或挤压
- 点击区域正常，没有“看得到但点不到”

### 游戏中 HUD
- 分数、鱼/金币、暂停按钮不进入刘海或挖孔区域
- HUD 不遮挡角色主要活动区域
- 不同分辨率下字体仍可读

### 暂停页 / 结算页
- 面板居中显示
- 按钮至少能完整显示并可点击
- 广告 Banner 若显示，不遮挡主操作按钮

### Google 登录
- 首页出现登录入口
- 点击按钮能触发日志和登录流程
- 登录成功后状态文本更新
- 进入游戏后登录入口不会遮挡核心操作区
